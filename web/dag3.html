Modul 5 - Hello World og Terminal/Konsoll

Hello World

Et Hello World program brukes ofte for å vise et kodeeksempel, eller et testprogram. Et slikt program inneholder det absolutte minimum av kode som er nødvendig for at programmet skal kjøre, og "Hello, World" skrives ut til konsoll eller annet sted på skjerm.

Hello World-program blir ofte veldig små, siden det sjeldent er mye som skal til for å skrive til skjerm. Det er derfor mest brukt for å illustrere den grunnleggende syntaksen programmeringsspråket har.

Eksempel i C og Java:

        // C
        #include <stdio.h>

          main( )
          {
                  printf("hello, world\n");
          }
      
        // Java
        public static void main(String[] args){
          System.out.println("Hello, World!");
        }

IDE

IDE står for Integrated Development Environment.

IDE-er er enkelt fortalt et program som gjør det lettere å programmere i. De inneholder som oftest en tekst-/kildekodeeditor med syntaksmerking, autofullføring av setninger, makroer (funksjoner som skriver forhåndsdefinert kode for deg), muligheter for å kompilere og kjøre programmer direkte og en debugger som enkelt lar deg gå igjennom koden din og sjekke om og hvor du har bugs.

Noen IDEer

Her er en liste med noen IDE-er:

  • Eclipse, hovedsakelig for Java.
  • NetBeans, hovedsakelig for Java.
  • IntelliJ, hovedsakelig for Java.
  • PhpStorm, hovedsakelig for PHP
  • WebStorm, hovedsakelig for JavaScript.
  • PyCharm, hovedsakelig for Python.
  • Atom, en teksteditor med masse konfigurasjonsmuligheter.

Det er fullt mulig å programmere, lage og kjøre kode uten å bruke IDE. Det er faktisk anbefalt for nybegynnere å benytte seg av teksteditorer og terminal/konsoll i starten, så man lærer seg det grunnleggende uten å bruke for mye hjelp i starten.

Terminal/Konsoll

Operativsystemer

Det finnes en god del operativsystemer (OS), men de aller mest kjente er Microsoft Windows, Apples iOS og macOS, og Linux.

Virkemåten til operativsystemene er mer eller mindre ulike. De har varierende forskjeller i arkitektur og oppbygging, andre instruksjonssett og kommandoer innad i systemene m.m.

Det er en stor, merkbar forskjell mellom Windows-operativsystemer og Mac/Linux. Dette er fordi Mac og Linux bygger på samme platform, de er begge Unix-baserte. Windows-operativsystemene bygger for øvrig på Windows NT som er skrevet i C og C++ (og noe i Assembly).

Terminal og Konsoll

Både terminal og konsoll kan kalles for kommandovindu/kommandolinje (eng. Command-line user interface (CLI)).

Kommandolinjen er et vindu der man kan skrive inn kommandoer til datamaskinen direkte.

På Mac og Linux bruker en oftest 'terminal', mens man i windows bruker 'konsoll' eller 'command prompt'/'cmd' (fra cmd.exe som er den kjørbare filen for kommandovindu på windows).

Windows sine kommandolinjespråk er PowerShell og Windows cmd, mens Mac og Linux bruker hovedsakelig Bash, et Unix-basert Shell (et grensesnitt for tilgang til operativsystemets tjenester).

Bruksområder

I teorien kan man bruke kommandovinduet til å utføre alle operasjoner et brukergrensesnitt (til operativsystemet) gir tilgang til.

    Eksempler:
  • Åpne programmer
  • Åpne, endre, lage nye, flytte på og slette filer
  • Skrive, lagre og endre tekst
  • Åpne nettsider
  • Bytte bruker, låse, slå av eller restarte datamaskinen

GUI og kommandolinje

Bruk av grafisk brukergrensesnitt (GUI), med musepeker, vinduer og visualiserte mapper og filer er kun en grafisk representasjon av hvilke kommandoer som kjøres underliggende på maskinnivå. Litt på samme måte er programmering en representasjon av maskinkode.

Eksempelvis i Finder (Mac) eller Filutforsker/Windows Explorer (Windows) så vil flytting, endring og åpning av filer ved bruk av musepeker bare sende et signal til maskinen at den skal kjøre de kommandoene med de gitte verdiene du som bruker velger. De samme kommandoene kan man selv skrive inn i kommandolinjen for å oppnå samme resultat, uten å bla seg frem og klikke på filen med musepekeren.

Noen kommandoer for UNIX (Mac/Linux m.m.)

  • cd - Change directory. Går inn i spesifisert mappe (directory)
  • ls - List. Lister opp filer og mapper i nåværende mappe
  • .. - Brukes med cd for å gå opp et nivå, altså en mappe tilbake
  • mkdir - Make directory, brukes for å opprette en mappe
  • open - Åpner en fil eller et program

Noen kommandoer for Windows (cmd)

  • cd - Change directory. Går til spesifisert mappe (directory)
  • .. - Brukes med cd for å gå opp et nivå
  • dir - Viser liste med filer og mapper
  • mkdir - Make directory. Lager ny mappe
  • rmdir - Remove directory. Sletter mappe
  • xcopy - Kopierer filer/mapper fra ett sted til et annet
  • start - Starter/åpner et program

Modul 6 - Intro til programmering i Java og PHP

Variabler

Som nevnt tidligere er programmeringsspråk en måte å fortelle datamaskinen hva som må gjøres for å komme frem til et ønsket resultat. Vi kan si at en oppskrift for å lage en brødskive krever at du skal ha:

  • 1 stk brød
  • 7.6 gram smør
  • 2 skiver ost
  • Ikke skinke

Altså, vi trenger å lagre disse verdiene når vi skriver algoritmen vår for å lage en brødskive, her kommer da konseptet 'variabler' inn. En variabel kan lagre data av typene heltall, desimaltall, tegn og tegnstrenger og bolske verdier (sann/falsk). Det er litt forskjellig hvordan man representerer en variabel i forskjellige programmeringsspråk. Vi skal først bruke programmeringsspråket Java for å se hva slags variabler man har der:

Variabler i java

Heltallstyper

I Java representerer man heltall ved 4 forskjellige typer: (navn, størrelse)

  • byte, 1 byte
  • short, 2 byte
  • int, 4 byte
  • long, 8 byte

Som du ser har disse forskjellige størrelser, talltypen 'byte' har størrelsen 1 byte, som vil si at den kan holde på 2^8 = 256 tall. Siden heltallstyper også skal kunne holde på negative tall, er den minste verdien til en byte -127, og den maksimale verdien 128 (den må også holde på tallet 0).

Desimaltallstyper

Desimaltall er mer anvendelige i situasjoner hvor man f.eks. i et banksystem hvor man ønsker å holde oversikt over saldo og transaksjoner. I Java har man to desimaltallstyper:

  • double
  • float

Tegn-verdier

Man trenger også variabler for å holde på tekst, som i java er:

  • char
  • String

En 'char' er en representasjon av én bokstav, mens en 'String' er en samling av flere bokstaver, altså en samling av flere 'char'. String brukes da naturligvis for å representere tekst.

Boolske verdier

En boolsk variabel er en variabel som kan ha to verdier. I Java kalles denne typen 'boolean', og har verdiene 'true' eller 'false'.

Deklarasjon av variabler i Java

Å deklarere en variabel betyr å lage en variabel når man programmerer - og gi den en verdi. Det er forskjellig hvordan man deklarerer en variabel, avhengig av typen til variabelen. Generelt så deklarerer man variabler med å skrive '<type> <navn> = <verdi>;'. Her er noen eksempler:

int alder = 23; double vekt = 90.4; char førsteBokstav = 'A'; String navn = "Andreas"; boolean erStudent = true;

Ser du nøye på koden over, så er det flere ting man må legge merke til. Det første er semikolon, som settes etter hver instruksjon man skriver, for å fortelle datamaskinen at dette er slutten av instruksjonen. Når man deklarerer en 'char' variabel, så må verdien dens settes mellom to enkle 'hermetegn', mens verdien til en String variabel settes mellom to doble "hermetegn". En siste ting er bruk av store og små bokstaver ved datatyper. 'String' er den eneste som ikke skrives med en liten bokstav først av variablene vi har sett på til nå - dette er fordi String ikke er en primitiv datatype - men et objekt (dette er et konsept som blir introdusert i studiet, så ikke tenk så mye på det nå 😃).

Etter man har deklarert en variabel (gitt den navn, type og evt verdi), så kan man endre på variabelen sin verdi på denne måten:

int a = 2; int b; b = a + 3;

Talltyper støtter også flere operasjoner som å addere (+), subtrahere (-), dividere (/), multiplisere (*) og modulo (%). Man kan også utføre '+' operasjoner med strenger (String variabler), og gjøres slik:

String fornavn = "Andreas"; String etternavn = "Strand"; String navn = fornavn + " " + etternavn; //navn har nå verdien "Andreas Strand"

Legg også merke til det som står etter '//' i forrige linje. Alt man skriver etter '//' blir ikke lest som kode av datamaskinen, men gir programmerer mulighet til å kommentere koden sin.
